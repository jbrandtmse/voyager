# Voyager FK Frame IDs

Auto-generated by `bake/src/fk_inventory.py` (Story 1.3 AC6). Reads `kernels/vg1_v02.tf` and `kernels/vg2_v02.tf` (both NAIF FKs) and emits the documented FRAME / TKFRAME entries that Stories 1.5+ rendering and Story 3.2 HGA-Earth synthesis depend on (per technical-research line 1531).

**Reproduce:** from the repo root,

    python bake/src/fk_inventory.py

## Frame ID conventions

Voyager NAIF frame IDs follow a `-3X000` pattern where `X` is `1` for Voyager 1 and `2` for Voyager 2. The 2-digit spacecraft body IDs (`-31` / `-32`) are SPK ephemeris IDs and are NOT frame IDs.

| Subsystem | V1 frame ID | V2 frame ID | Parent / class |
|---|---|---|---|
| Spacecraft bus | -31000 | -32000 | CK class 3 (inertial via CK) |
| Scan platform | -31100 | -32100 | CK class 3 (inertial via CK) |
| NA camera (ISSNA) | -31101 | -32101 | TK class 4 — relative to SCAN_PLATFORM |
| WA camera (ISSWA) | -31102 | -32102 | TK class 4 — relative to SCAN_PLATFORM |
| PPS | -31103 | -32103 | TK class 4 — relative to SCAN_PLATFORM |
| UVS | -31104 | -32104 | TK class 4 — relative to SCAN_PLATFORM |
| UVSOCC | -31105 | -32105 | TK class 4 — relative to SCAN_PLATFORM |
| IRIS | -31106 | -32106 | TK class 4 — relative to SCAN_PLATFORM |
| IRISOCC | -31107 | -32107 | TK class 4 — relative to SCAN_PLATFORM |
| CONE_CLOCK | -31200 | -32200 | TK class 4 — relative to SC_BUS |
| AZ_EL | -31300 | -32300 | TK class 4 — relative to SC_BUS |
| **HGA** | **-31400** | **-32400** | TK class 4 — relative to SC_BUS (boresight +Z) |


**Note on Story 3.2 / FR18:** the HGA frames are fixed offsets relative to the spacecraft bus (`VG1_SC_BUS` / `VG2_SC_BUS`). When the bus attitude is provided by `vgr{1,2}_super_v2.bc` (CK class 3), the HGA boresight in inertial coordinates is fully determined. AttitudeService can use the +Z axis of the HGA frame as the HGA-Earth pointing reference.
