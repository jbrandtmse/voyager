## Findings + remediation

**Critical / serious findings (block next milestone):** NONE.

**Moderate / minor advisories surfaced by axe-core route suite:**

- `landmark-one-main` (moderate) — `<main>` landmark missing on the simulation surface (`/` and `/c/<slug>` chapter routes). The simulation surface is a canvas-based experience without a semantic document body; the HUD overlay uses `<aside>` (landmark-banner adjacent) but no `<main>`. **Disposition:** DEFERRED to Epic 7 polish — adding `<main>` requires deciding whether to wrap the canvas + overlays (semantic ambiguity) or to use ARIA `role="main"` on the canvas itself. Not a Story 6.4 in-scope fix.
- `page-has-heading-one` (moderate) — `<h1>` missing on the simulation surface. The simulation isn't a document; the chapter title in `<v-hud-chapter-title>` is an `<h1>` semantically but is announced via `aria-live` mirror rather than rendered as a literal `<h1>` (HUD style discipline). **Disposition:** DEFERRED to Epic 7 polish — needs a UX decision about whether to expose a visually-hidden `<h1>` for SR consumption.
- `region` (moderate, 3-4 nodes per route) — some HUD chrome elements live outside a recognised landmark. **Disposition:** DEFERRED with the same Epic 7 routing as `landmark-one-main`.

**Why these are advisories, not blockers:** the Voyager simulation surface is intentionally a canvas-based interactive experience, not a document. The axe-core landmark heuristics are calibrated for traditional document layouts. The narrative chapter title IS spoken by screen readers (via `<v-hud-chapter-title>`'s aria-live mirror), so the information is reachable — just not via the document-landmark mechanism axe-core expects. Per Story 6.4 AC2: "moderate and minor violations are reported in CI output but do NOT block the build (logged + linked to the routes affected; the CI workflow file emits them as warnings)." That contract is honored here.
