### Local-env constraint

Some lead environments do not have `uv` / SpiceyPy installed (see the
Story 3.4 deferred-work `[3.4 / LOW]` item). In those environments the
Python script cannot run; the Vitest test detects the missing fixture
via `fs.existsSync` and `describe.skipIf(!fixturePresent)` the entire
suite. The rest of `cd web && npm test` continues to pass. The full
gate runs in CI.

### Deterministic-rerun contract (NFR-R4)

The fixture is byte-stable across reruns when the inputs (CK + FK +
SCLK + LSK kernels) are byte-stable. Determinism comes from:

- Fixed RNG seed (`RNG_SEED = 42`, same as the L1 validator).
- Per-window seeds derived as `RNG_SEED + window_index * 2 + spacecraft_offset`
  so each (window, spacecraft) draws independently and re-runs reproduce
  the same ET sequence.
- Records sorted by `(spacecraftId, et)` ascending before serialization.
- `json.dumps(sort_keys=True, indent=2)` for diff stability.
- No host-clock reads.

### Regeneration trigger

The fixture must be regenerated when any of these change:

- `kernels/vgr1_super_v2.bc`, `kernels/vgr2_super_v2.bc`, or any other CK.
- The FK frame kernels (`kernels/vg1_v02.tf`, `kernels/vg2_v02.tf`).
- The SCLK kernels.
- The LSK (leap-second kernel).

Story 7.x's planned kernel-drift-report workflow will reference this
regeneration trigger and run the script as part of its drift-detection
flow. For now the trigger is honored manually + automatically in CI.

### Size cap + LFS routing

The fixture is committed to the repo as test data (per Story 3.7 AC4).
Size cap: â‰¤ 2 MB. The Python script halves the per-window sample count
once and re-emits if the cap is breached; beyond a single halving the
script exits non-zero so a regression surfaces in CI rather than
silently shrinking the fixture below a useful sample density. If the
fixture eventually exceeds the cap durably, route it via Git LFS per
ADR-0011 by extending `.gitattributes` with a `filter=lfs` rule for
the JSON path.
