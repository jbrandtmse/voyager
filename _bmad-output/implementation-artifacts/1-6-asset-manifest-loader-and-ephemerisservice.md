# Story 1.6: Asset Manifest Loader and EphemerisService

Status: done

## Story

As a visitor,
I want the simulation to fetch trajectory binaries on demand and interpolate spacecraft and body positions at the current ET,
so that the time scrubber in Story 1.9 has real data to drive the scene, and FR55 (L2 hook) and NFR-P7 are operational.

## Acceptance Criteria

**AC1 — `ManifestLoader.load()` fetches and validates `manifest.json`:**
- **Given** the bake pipeline has produced `bake/out/manifest.json` and VTRJ files, and `just copy-bake-to-web` has copied them to `web/public/data/manifest.json` and `web/public/data/*.bin.br`,
- **When** the web app boots and calls `ManifestLoader.load(url = '/data/manifest.json')`,
- **Then** the loader fetches the manifest, validates its schema with `zod`, and returns a typed `Manifest` object exposing accessors for `bodies[].files[]` entries with the runtime shape from architecture Decision 1b: `{ naifId, name, files: [{ timeRangeEt, cadenceSec, kind, url, sha256, sizeBytes }] }`,
- **And** an invalid schema (missing required fields, unknown `schemaVersion` major, wrong types) fails fast: `ManifestLoader.load()` rejects with a `ManifestValidationError` whose message identifies the offending path (e.g., `bodies[0].files[2].sha256: required`),
- **And** the loader is callable multiple times and returns the same parsed `Manifest` for the same URL (cached at module scope; the second call resolves synchronously to the cached value).

**AC2 — `ChunkLoader` fetches and decodes VTRJ binaries with an LRU cache:**
- **Given** the manifest is loaded,
- **When** `ChunkLoader.load(file: ManifestFile)` is called for the first time on a given `url`,
- **Then** the loader:
  1. Fetches the URL via `fetch()` (no auth, no special headers; same-origin)
  2. Decodes the brotli stream via `DecompressionStream('br')` into an `ArrayBuffer`
  3. Verifies the body's SHA-256 against the manifest's `sha256` field; on mismatch, rejects with a `ChunkIntegrityError` naming the file and both SHAs
  4. Parses the 40-byte VTRJ header (matching Story 1.4's writer layout: magic="VTRJ", version=1 u16 LE, body_id i32 LE, et_start f64 LE, et_end f64 LE, sample_count u32 LE, cadence_seconds f64 LE, reserved 2 bytes)
  5. Constructs a `Float64Array` view over the body (`sample_count × 6` doubles for state vectors)
  6. Returns a `LoadedChunk { header: VtrjHeader, samples: Float64Array }` and stores it in an LRU cache (default capacity 12 chunks — enough for V1 7 segments + V2 11 segments combined; the LRU size is a tunable constant in `chunk-loader.ts`)
- **And** subsequent calls with the same `url` return the cached chunk synchronously (no network access; observable via a per-test mock that asserts `fetch` was called exactly once)
- **And** the loader exposes a `loading` boolean (initially `false`, `true` while any fetch is in flight) and a `subscribe(cb: (loading: boolean) => void): () => void` observer pattern — this is the hook future `ClockManager` will use for FR7 auto-speed-cap.

**AC3 — `EphemerisService.getStateAt(et, bodyId)` returns interpolated state:**
- **Given** `ManifestLoader` and `ChunkLoader` are both wired,
- **When** code calls `EphemerisService.getStateAt(et: number, bodyId: number): { position: WorldVec3, velocity: WorldVec3 } | null`,
- **Then** the service:
  1. Finds the manifest entry for `bodyId` (V1 = -31, V2 = -32 for this story; planets in Story 1.13)
  2. Identifies which segment file covers `et` (binary search over the `files[].timeRangeEt` ranges)
  3. If the chunk is not yet loaded, calls `ChunkLoader.load(file)` and returns `null` to signal "data not yet available" (per architecture line 874: never throw, never block — `null` means "loader is fetching, try again next frame")
  4. If the chunk is loaded, performs cubic Hermite interpolation over position+velocity for the given `et`, returning the interpolated state as `{ position: WorldVec3, velocity: WorldVec3 }`
- **And** the service is referentially transparent (same `(et, bodyId)` always returns same value once the chunk is loaded)
- **And** the service has a convenience `getPosition(et, bodyId): WorldVec3 | null` and `getVelocity(et, bodyId): WorldVec3 | null` returning only position or velocity respectively
- **And** an `et` outside any segment range returns `null` (no extrapolation)

**AC4 — Cubic Hermite interpolation accuracy holds at runtime (NFR-P9 L2 hook):**
- **Given** the L1 Python harness already validated bake-time accuracy in Story 1.4 (max ≤ 20 km / RMS ≤ 5 km),
- **When** `EphemerisService.getStateAt` is queried at a deterministic set of ETs (test-fixture set: 4-hour grid sampled across each segment),
- **Then** the returned positions match a pre-computed SpiceyPy reference set (committed as a test fixture at `bake/out/l2-reference-fixtures.json` — generated by a small Python script in this story) to within 20 km max / 5 km RMS,
- **And** the L2 fixture generator is `bake/src/generate_l2_fixtures.py` (new): samples a deterministic ET grid, calls SpiceyPy, writes a JSON file with `{ etArray, expectedStatesPerBody }` for the web-side test to load.
- **Note:** the full per-frame L2 harness (Story 3.7) does this comparison across the entire mission timeline in CI; this AC4 is the smaller "smoke test" that proves the runtime interpolator matches bake-time accuracy on a sample grid.

**AC5 — Interpolation perf hits NFR-P7 (≤ 1 ms median):**
- **Given** the runtime,
- **When** the developer navigates to `?perf=ephemeris` (the dev-mode perf route; mirrors Story 1.5's `?dev=precision` pattern),
- **Then** a perf harness runs in the browser:
  1. Loads all VTRJ chunks for V1 + V2 (the only bodies available in this story — Story 1.13 adds the planets)
  2. Calls `EphemerisService.getStateAt(et, bodyId)` for both spacecraft in a tight loop of 1000 iterations at varying ETs
  3. Measures via `performance.now()` and reports median, p95, p99 per-call costs
- **And** the median cost is ≤ 1 ms when measured on the developer's machine (NFR-P7 target is 1 ms for 12 bodies; for 2 bodies the target is proportionally smaller — assert ≤ 0.2 ms median for 2 bodies as the equivalent gate)
- **And** the perf harness output is rendered to a `<pre>` block on the page (no rich UI; just text)
- **And** a NOTE on the page documents that the 12-body NFR-P7 measurement re-runs in Story 1.13 once celestial bodies are added.

**AC6 — `just copy-bake-to-web` is implemented (not stub):**
- **Given** Story 1.4 left `just copy-bake-to-web` as a no-op stub,
- **When** the recipe is invoked,
- **Then** it copies `bake/out/manifest.json` → `web/public/data/manifest.json` and `bake/out/*.bin.br` → `web/public/data/*.bin.br` (preserving filenames),
- **And** the recipe is idempotent (re-running with unchanged inputs produces no diff in the destination tree),
- **And** the recipe creates `web/public/data/` if it doesn't exist,
- **And** a small Python script `scripts/copy_bake_to_web.py` does the actual copy (so the recipe is platform-portable — no `cp -r` or `robocopy` that diverge between OS); the justfile recipe just wraps the script.

## Dev Notes

### Architectural Compliance — load-bearing constraints

- **Float64 on the world-space side, ALWAYS.** Architecture lines 79, 359, 976. `WorldVec3` is Float64-backed. The cast to Float32 happens ONLY at the floating-origin recenter site (Story 1.5's `renderVec3FromWorld`). EphemerisService outputs `WorldVec3` (Float64).
- **Cubic Hermite over position + velocity.** Architecture line 308, ADR 0003. The hermite math goes in `web/src/math/hermite.ts`.
- **`null` for missing data, NEVER throw in hot path.** Architecture line 874: "if a chunk is missing, the service returns nearest-available data and `ChunkLoader` emits `loading: true`." For this story, we return `null` (a simpler signal than "nearest-available" — that's a Story 1.10 + 1.11 refinement once `ClockManager` is in place to time-travel back to the last valid frame).
- **`ChunkLoader` is the ONLY async boundary in the service graph.** Architecture line 874, 1221. `EphemerisService.getStateAt` is sync.
- **VTRJ format is locked to Story 1.4's writer.** 40-byte LE header + Float64Array body, brotli-compressed. The web-side decoder must round-trip exactly.
- **`web/public/data/manifest.json` is the runtime contract.** Architecture line 263, 992, 1213. The bake writes; the web reads.
- **Subscribe/unsubscribe pattern.** Architecture line 1220 names this as the canonical observable surface.

### Browser API requirements (browser tier 1)

- **`DecompressionStream('br')`** — required for brotli decode. Chrome 120+, Firefox 126+, Safari 17.5+. Pre-2024 browsers will need polyfill OR the fallback page (Story 1.8). For Story 1.6, just rely on the native API; the fallback page catches unsupported browsers.
- **`crypto.subtle.digest('SHA-256', ...)`** — required for chunk integrity. Universally supported.
- **`fetch` + `Response.arrayBuffer()`** — universally supported.

### Latest Tech Information

- **Zod 3.x** is the canonical TypeScript schema validator. ~12 KB minified. No new transitive deps beyond TypeScript's own.
- **`DecompressionStream('br')`** is the modern, standards-track brotli decoder. Available in all browsers that meet the FR50/FR51 baseline.
- **`crypto.subtle.digest`** is part of WebCrypto, universally supported in browser contexts (and Node 16+).
