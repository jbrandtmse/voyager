## What went well

### 🏆 Top win (Project Lead's selection): Story 6.4's axe-core surfacing + fixing 3 silent a11y defects

The expanded axe-core gate caught 3 critical/serious violations that the prior 1-test-per-component baseline missed:

1. **`aria-valuenow=<ISO>`** — `<v-timeline-scrubber>` shipped this for 4 epics. Story 6.4's component-state-matrix axe check flagged it on the first run.
2. **Orphan `role="row"`** — `<v-hud-instruments>` had this since Story 2.9. Same situation: shipped silently across 4 epics.
3. **`aria-hidden=true` + focusable content** — three components (`<v-help-overlay>`, `<v-chapter-index>`, detail-variant `<v-timeline-scrubber>`) all had this; fix is paired `aria-hidden` + `inert`.

These would have shipped silently without Story 6.4's gate. Production users (especially screen-reader users) would have hit real issues. The lesson is **expand discipline early; don't trust per-component-default-state tests as full coverage.**

This is also the canonical Rule 9 / ADR-0025 closure moment — extracting `dialog.ts` as the third primitive completes the ADR's obligations.

### Other wins worth recording

- **Rule 14 firing self-referentially on Story 6.0** — the spec-arithmetic rule's first real-world application was inside the dev cycle for the story that added the rule. Caught two genuine spec-vs-implementation drifts (CSS asset prefix + FOUC-shim source) at planning-review time. Satisfying loop closure.
- **+532 net tests across 7 stories** — largest test-pyramid growth of any epic. Each story added discipline-tier defense tests (cross-reference rot defense, doc-integrity grep checks, regression-case rewrites).
- **Story 6.6 code review was 0 HIGH / 0 MED / 0 LOW clean** — the cleanest review of the epic, suggesting the discipline accumulated through Stories 6.0–6.5 paid off by the time the final polish story shipped.
- **Story 6.3 caught the Story-6.0↔Story-6.2 interaction defect** — QA root-caused the build-dist-layout failure to Story 6.2's defensive corner CSS supersession; CR rewrote inline to delta-based detection. Three-story-deep integration coherence held, but only barely; see "What didn't go well" #1.
- **Placeholder-fallback posture worked twice cleanly** — Story 6.1 audio + Stories 6.5/6.6 docs all shipped with documented "real artifact requires follow-up commit by maintainer" callouts. No surprise gaps; no fake completeness.

## Lessons applied + new lessons surfaced

### Epic 5 lessons that paid off in Epic 6

- **Rule 5 in-place amendments** — 2 amendments across Stories 6.0 + 6.1. Same discipline as Epic 5's 3 amendments. Zero "code comment + deferred-work" workarounds.
- **Rule 10 Lit declare+ctor pattern** — every new Lit component in Epic 6 (`<v-audio-toggle>` in 6.1, narrow-viewport-toggle-button-in-`<v-hud>` in 6.2, `⋯` button in 6.2) used the canonical declare+ctor pattern. Zero Rule 10 violations surfaced.
- **Rule 11 build-pipeline E2E tests** — Story 6.6's L4 mask + tolerance tightening is the kind of pipeline-integration test Rule 11 mandates. Pattern carried forward.
- **Lesson 8 per-story smoke** — every story's lead smoke caught at least documentation gaps + verified end-to-end wiring. No story shipped without a documented smoke evidence file.
- **Cross-review discipline (Epic 5 retro Lesson 4)** — Story 6.3 code review caught the 6.0↔6.2 interaction defect; this is the cross-cutting-coherence value cross-review surfaces that per-story smokes can't.

### New lessons from Epic 6

1. **Expanded automated discipline catches silent defects across multiple epics.** Story 6.4's axe-core component-state-matrix expansion (1-default-state → multi-state per component) surfaced 3 critical/serious a11y violations that had shipped for 4 epics. **Lesson:** widening automated test surface periodically — not just at initial baseline — catches accumulated drift. Action: Epic 7 should include a similar periodic-expansion pass for performance NFRs (60 FPS, ≤3s TTI, ≤35 MB first-paint, ≤150 MB total).

2. **Self-referential rule firing is satisfying loop closure.** Rule 14 (spec arithmetic) was added by Story 6.0 dev; that same dev cycle's planning-review phase caught two genuine spec drifts that Rule 14 was designed to prevent. **Lesson:** when adding a process rule, the rule's first application should ideally be on the same story that authored it — the loop closes and the rule's value is demonstrated immediately.

3. **Placeholder-fallback posture works when the procurement path is documented + auditable.** Story 6.1 placeholder audio + Stories 6.5/6.6 doc-template posture both shipped cleanly because the docs explicitly stated "real artifact deferred; here's the protocol." Future story authors facing similar procurement-or-execution gaps should use the same posture — never ship "complete" with fake completeness; always ship "documented-as-placeholder" with explicit follow-up protocol.

4. **Cross-story integration coherence is fragile beyond 2 stories.** The 6.0↔6.2↔6.3 chain caught the build-dist-layout defect but only at story 3. The two-story window (6.0→6.2) was insufficient. **Lesson:** X.0 regression-test files need explicit invariants so downstream stories can verify they don't break the invariant before commit. Action #2 (Rule 18).

5. **"Lint baseline preserved" can become a euphemism for "ignored."** Every Epic 6 story preserved the 4-warning lint baseline without addressing them. The discipline of "don't make it worse" was held, but the discipline of "make it better" was never triggered. **Lesson:** if a baseline persists more than 2 epics unchanged, surface it explicitly as an "address-or-accept" decision point. Action #4.

6. **Story-to-story cycle-log authorship needs explicit convention.** Multiple Epic 6 dev agents pre-populated `dev_complete` cycle-log entries; the workflow spec implies the lead writes the cycle log. No semantic divergence resulted, but the boundary blurred. Action #5.
