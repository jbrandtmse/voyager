## 5. Decisions Made During Epic 2

### Decision A — Chrome-vs-editorial split is the embed-mode contract

The deciding question for every new top-level component: **"Is this content the museum kiosk needs to show, or navigation around the content?"**

Chrome (skipped in embed mode):
- `<v-chapter-index>` toggle (Story 2.3)
- Attributions footer (Story 2.7)
- `<v-help-overlay>` toggle (Story 2.8)

Editorial / simulation (kept in embed mode):
- HUD + `<v-hud-instruments>` (Stories 1.11, 2.9)
- `<v-timeline-scrubber>` + markers (Stories 1.9, 2.2)
- `<v-chapter-copy>` text-cards (Story 2.9)
- `<v-play-button>`, `<v-speed-multiplier>` (Story 1.10)

Implementation pattern: **skip-mount at first-paint** (NOT `display: none`). Keyboard shortcuts that target absent chrome become no-ops via the no-listener-attached pattern.

**Status:** Observed and applied across 4 stories; not yet promoted to a formal voyager-skill-rule. Carried as observation for `/epic-cycle` v2 design pass.

### Decision B — cr-2-7's "no pushState before location.assign" pattern

Cross-pathname navigations (e.g., `/` → `/about`, `/about` → `/c/<slug>`) MUST use `window.location.assign(url)` directly, **without** a preceding `history.pushState()`. The push-then-assign combination is a same-document hash navigation in real browsers (no reload) and silently keeps the previous surface mounted. happy-dom stubs `location.assign` so unit tests do not catch this — the bug is real-browser-only.

**Status:** Pattern was applied in Story 2.7 (footer link) and Story 2.8 (`A` shortcut). cr-2-8 explicitly enforced the inheritance.

### Decision C — Editorial content stays mounted in embed mode

`<v-chapter-copy>` (Story 2.9) MOUNTS in embed mode. Rationale: text-cards are editorial content, not chrome. A museum kiosk wants the V1/V2 heliopause text-cards visible. This is the inverse of the chapter-index/help-overlay/footer chrome-skip rule and is the disambiguating example for the "kiosk content vs navigation around content" question.

## 7. Key Lessons (for future epics)

1. **Code review IS the last line of defense for real-browser bugs.** 5 of 6 HIGH-severity catches in Epic 2 were happy-dom blind spots. The lead-executed Chrome DevTools MCP smoke (ADR-0010 Layer 1) closed the loop, but CR's auto-resolution authority is what kept defects from accumulating to the smoke gate.

2. **Pattern inheritance across stories propagates without manual coordination.** cr-2-7 → cr-2-8 is the clearest example: cr-2-7 invented the no-pushState pattern, cr-2-8 enforced it as a check on dev-2-8's `A` shortcut implementation. The discipline travels.

3. **The chrome-vs-editorial split is the museum-kiosk contract.** If we'd been less clear about this in Story 2.5, the help-overlay (2.8) and attributions footer (2.7) decisions would have been ambiguous. The early decision in 2.5 set the rule for the rest of the epic.

4. **3-layer parity tests work.** Story 2.6 (OG ↔ ALL_CHAPTERS) and Story 2.9 (mission-facts.ts ↔ MISSION_FACTS.md) both used the unit + FS + integration parity pattern. The R4-style mitigation generalizes.

5. **Story X.0 cleanup pattern is non-negotiable.** Story 2.0 closed 11 Epic 1 deferred items + 3 retro action items before substantive Epic 2 work started. The cycle log shows zero "deferred items silently accumulating" — exactly the failure mode the Epic 1 retro identified.
