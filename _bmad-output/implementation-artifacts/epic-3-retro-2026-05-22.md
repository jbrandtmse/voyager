## What went well

### 1. The v3 `/epic-cycle` workflow as a process win

**The user's headline reflection.** The v3 workflow replaced v2's Agent Teams + `SendMessage` + `STATUS: completed` envelope (whose silence-recovery rate climbed toward 100% in v2 production) with plain `Agent` tool calls returning structured closing-summary sections inside the tool's natural return value. The lead reads `## Files Modified` / `## Tests Added` / `## Decisions` / `## Issues Encountered` directly from each agent's return; falls back to `git status --short` only if a section is missing.

Across 9 commits this epic, the closing-summary sections were present every time. No envelope silence. The fallback path (lead reconstructing from `git status`) was never used. This represents a structural shift from "primary path is unreliable; fallback IS the normal flow" to "primary path is the normal flow; fallback is a true defense-in-depth."

The lead-side ADR-tooled gate (Layer 1; moved off subagents per Rule 7) + the per-story-smoke gate (Rule 3) collectively caught 6 HIGH defects this epic — empirical justification for keeping both gates load-bearing.

### 2. The differentiator works end-to-end

Story 1.12's single-LOD Draco-compressed flat-mesh spacecraft + Story 1.6's manifest-loading shell were ALL that existed pre-Epic-3. By end-of-epic, the visitor sees: spacecraft physically articulating per CK-reconstructed (or synthesized HGA-Earth-pointing) attitude; a thin amber wireframe cone extending from the scan platform showing where the NA camera was pointed; a quiet "ATT ● CK reconstructed" or "ATT ● Synthesized (HGA Earth-pointing)" HUD label that never lies about the substrate. The PRD's differentiator promise is now structurally real, not a planning aspiration.

## What didn't go well

### 1. Local toktx install + the cascade of build_glb.ts defects only smoke could catch

**The user's headline pain point.** Story 3.3 required the user to manually accept a Windows UAC prompt to install Khronos KTX-Software (the harness blocked elevation; user clicked through the installer). Once toktx was on PATH and the lead ran `npm run build-glb`, two HIGH defects surfaced sequentially: WebP→PNG transcode missing (sharp not invoked despite being a transitive dep) and `MeshoptEncoder` missing from the NodeIO dependency registry. 22 dev-tier unit tests had passed cleanly because the full pipeline was never exercised end-to-end pre-smoke. **Build-pipeline scripts need end-to-end runtime tests, not just unit-tier piece tests.**

### 2. The chunk-loader chapter-route bug had been silent for 6 stories

Story 2.4's `/c/<slug>` URL scheme silently broke chunk-loader's relative URL resolution. Stories 2.5–3.2 navigated to chapter routes via the chapter index but never specifically exercised the chunk-loader from that context. Story 3.3's smoke caught it only because the AC9 probe sequence navigated to `/c/v1-jupiter` AFTER homepage. **A defect that lives across 6 stories has compounding cost; Rule-3 smokes catch this class faster than test-tier additions can.**

### 3. The Rule 5 V2 Saturn cadence tripwire is canonical but uncomfortable

Story 3.7's L2 validator did exactly its job — surfaced a real NFR-P10 budget violation in already-committed Story 3.1 work. The user chose the canonical Rule 5 outcome (gate intact, defer upstream amendment to hotfix). CI will be RED on first push until the Story 3.1 cadence-amendment hotfix lands. This is the system working as designed, but it means Epic 4 dev can't kick off with green CI until the hotfix lands. **The cost of catching the defect late (Story 3.7 rather than during Story 3.1's own smoke) is the CI-RED interregnum.**

### 4. Local-env constraint cascade: uv + toktx limitations on the lead's environment

Multiple stories ended with the smoke verifying only the synthesized path (no CK files locally because `just bake-attitude` needs uv). Stories 3.4, 3.5, 3.6 all noted in their smoke evidence that the CK-articulation visual verification is deferred to post-CI-bake artifacts. This is acceptable defense-in-depth (unit + integration tier cover the CK path) but it does mean the visual gate fires late. **Filed as `[3.4 / LOW]` in deferred-work; the natural landing is Epic 4's first encounter chapter.**

### 5. Lit class-field shadowing tripped Story 3.6 hard

19 of 25 unit tests failed on first run with a Lit runtime warning pointing at `lit.dev/msg/class-field-shadowing`. Resolved via `declare` + constructor init (mirrors `v-chapter-index.ts`). **The project should encode the `declare` + ctor-init pattern in a project-context note** so future Lit-component stories don't relearn this.

## Significant discoveries (epic update implications)

**Story 3.1 cadence calibration is not encounter-agnostic.** The 5-sec uniform cadence Story 3.1 settled on was calibrated against V2 Uranus slew rates (~0.34 mrad knot-to-knot). V2 Saturn at peak slew is 5.8 mrad knot-to-knot — 17× higher angular rate, breaching NFR-P10 by ~3.6×. Story 3.1's planning artifact amendment at epics.md line 1281 documented the calibration explicitly against V2 Uranus only; the implicit assumption that other encounters would behave similarly was unverified.

**Epic update needed:** Story 3.1 epic spec needs a cadence-amendment block recording the per-encounter cadence calibration. The Story 3.1 hotfix that lands Action #1 is the natural place to make the amendment in place (Rule 5).

**No other planning-artifact amendments needed.** PRD/architecture are sound across the epic. The 2 ADR additions (0028 — wide-angle deferral) are clean.

## Process lessons (carry forward to future epics)

1. **The per-story-smoke gate is the single highest-leverage process check.** 6 HIGH defects caught this epic that all four automated test tiers shipped green. Test pyramid is necessary but not sufficient — Rule 3 is the bridge from "tests pass" to "the product works."

2. **Build-pipeline scripts especially need end-to-end runtime tests.** Story 3.3's build_glb.ts had 22 unit tests that passed without the full pipeline running. Two HIGH defects only surfaced when the lead ran the pipeline end-to-end. The unit-tier piece tests are insufficient evidence on their own for any script that calls out to external binaries (toktx, sharp, etc.) or chains multiple library calls.

3. **Smoke catches across-story defects faster than test-tier additions.** The chunk-loader chapter-route bug was pre-existing since Story 2.4 but never surfaced because no smoke had specifically navigated to a chapter route from the chunk-loader's perspective. The Rule-3 smoke pattern naturally varies the navigation surface across stories, catching cross-story defects.

4. **Rule 5 NFR tripwire response — gate intact, amend planning artifact in place, defer the fix — is the right canonical outcome.** Story 3.7's V2 Saturn finding tested this exact path. The CI-RED interregnum is the price of catching the upstream defect at a late gate; the alternative (soft-gate or remove the assertion) would defeat Rule 5.

5. **`/epic-cycle` v3 dropped the Agent Teams + SendMessage envelope; closing-summary sections inside the tool return have been 100% reliable across 9 commits.** This is the structural improvement the user called out.

6. **The user's authorization on substantive procurement decisions (toktx install, NFR-P10 gate intact) was load-bearing.** The workflow's `AskUserQuestion` gate at clarification points worked exactly as designed — the user made the right calls at each decision point.
